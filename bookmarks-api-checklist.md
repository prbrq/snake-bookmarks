# Bookmarks API — Критерии готовности

## 1. Настройка проекта

- [x] Проект запускается одной командой (`uvicorn main:app --reload` или аналог)
- [x] Зависимости зафиксированы в `requirements.txt` или `pyproject.toml`
- [x] Используется SQLite (для простоты, без внешних сервисов)
- [x] При первом запуске база и таблицы создаются автоматически

## 2. Модели и миграции

- [x] Таблица `bookmarks` — id, url, title, description (nullable), created_at
- [x] Таблица `tags` — id, name (уникальное)
- [x] Таблица связи `bookmark_tags` — bookmark_id, tag_id
- [x] Связь many-to-many работает корректно

## 3. CRUD закладок

- [x] `POST /bookmarks` — создаёт закладку, принимает `{ url, title, description?, tags?: string[] }`
- [x] Если переданный тег не существует — он создаётся автоматически
- [x] Дублирующийся URL возвращает `409 Conflict`
- [x] `GET /bookmarks` — возвращает список закладок с вложенными тегами
- [x] `GET /bookmarks/{id}` — возвращает одну закладку; `404` если не найдена
- [x] `PUT /bookmarks/{id}` — обновляет закладку (включая теги); `404` если не найдена
- [x] `DELETE /bookmarks/{id}` — удаляет закладку; `404` если не найдена

## 4. Фильтрация и пагинация

- [x] `GET /bookmarks?tag=python` — фильтрация по тегу
- [x] `GET /bookmarks?limit=10&offset=0` — пагинация
- [x] `limit` по умолчанию = 20, `offset` по умолчанию = 0
- [x] Ответ содержит `total` (общее количество) для построения пагинации
- [x] Сортировка по `created_at` от новых к старым

## 5. Поиск

- [x] `GET /bookmarks/search?q=текст` — ищет по `title` и `description`
- [x] Пустой запрос `q` возвращает `400 Bad Request`
- [x] Поиск регистронезависимый
- [x] Результаты поиска тоже поддерживают пагинацию

## 6. Теги

- [ ] `GET /tags` — возвращает список тегов с полем `bookmarks_count`
- [ ] Теги без закладок тоже отображаются (с `bookmarks_count: 0`)
- [ ] Имена тегов хранятся в нижнем регистре (нормализация)

## 7. Валидация

- [ ] `url` проверяется на корректность (валидный URL)
- [ ] `title` не может быть пустой строкой
- [ ] Некорректные данные возвращают `422` с понятным сообщением

## 8. Формат ответов

- [ ] Все ответы в JSON
- [ ] Даты в формате ISO 8601
- [ ] Ответ на создание возвращает `201 Created`
- [ ] Ответ на удаление возвращает `204 No Content`
- [ ] Структура ответа списка: `{ "items": [...], "total": N, "limit": N, "offset": N }`

## 9. Документация

- [ ] Swagger UI доступен на `/docs` (FastAPI генерирует автоматически)
- [ ] Все эндпоинты отображаются с описаниями
- [ ] Схемы запросов и ответов видны в документации

## 10. Финальная проверка

- [ ] Создать 5+ закладок с разными тегами
- [ ] Проверить фильтрацию по тегу — возвращает только нужные
- [ ] Проверить пагинацию — limit/offset работают корректно
- [ ] Проверить поиск — находит по части слова в title и description
- [ ] Удалить закладку — проверить что теги не удалились
- [ ] Попробовать создать закладку с дублирующимся URL — получить 409
- [ ] Попробовать получить несуществующую закладку — получить 404
